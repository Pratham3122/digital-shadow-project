Understanding Each Step

▶ Start & Welcome

The program initialises global pointers (head = NULL, stack_top = NULL) and prints a styled ASCII welcome banner before entering the main menu loop.

📋 Display Menu

A formatted menu of 10 operations (1–9 and 0) is printed. The user enters their choice which is read via scanf and passed to a switch statement.

◆ Read Choice

The decision diamond branches execution to the chosen operation. Invalid inputs fall through to a default error message before looping back.

➕ Add Event

Allocates a new node with malloc(), fills in the event name, category, and auto-generates the timestamp. The node is appended at the tail of the linked list and pushed onto the undo stack.

🗑️ Delete Event

Searches for the node by ID, adjusts the previous node's next pointer to skip the target, and calls free() to release memory. Handles head, middle, and tail deletions.

✏️ Edit Event

Traverses the list to find the node by ID, then prompts for a new name and category. The existing fields are overwritten in place — no reallocation needed.

🔍 Search Event

Performs a case-insensitive partial match using strstr() + tolower(). Every matching node is displayed, providing Google-like fuzzy search behaviour.

📋 Display Timeline

A simple head-to-tail traversal printing each node's ID, name, category, date, and time in a formatted table row.

📅 Today's Timeline

Gets the current date and filters the linked list, displaying only nodes whose date matches today.

🔁 Replay Day

Pushes every event onto a temporary stack, then pops and prints them all. This demonstrates the LIFO property — events appear in reverse chronological order.

↩️ Undo Event

Pops the most recent event from the undo stack and deletes the corresponding node from the linked list by its ID.

📊 Statistics

Single-pass aggregation: counts total events, today's events, category frequencies, and identifies the most frequent category. Also displays the first and last events.

🚪 Exit & Cleanup

Traverses both the linked list and the stack, calling free() on every node to prevent memory leaks. Then prints a goodbye message and returns 0.
