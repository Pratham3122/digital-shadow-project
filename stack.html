Understanding Stacks

The LIFO powerhouse behind undo operations and reverse-chronological replay in Digital Shadow.

Fundamentals

What is a Stack?

A stack is a linear data structure that follows the Last In, First Out (LIFO) principle — the most recently added element is the first one removed.

📱 Phone Unlocked
🚶 Left Home
🏫 Entered College
📶 Connected WiFi
← TOP ← BOTTOM
📚 LIFO — Last In, First Out

Think of a Stack of Plates

Just like a stack of plates in a cafeteria — you can only take the top plate (pop) and add a new plate on top (push). You cannot access plates in the middle without removing the ones above.

In Digital Shadow, when you "replay" your day, events are pushed onto a stack in chronological order. When popped, they come out in reverse — showing your day backwards, from the most recent event to the earliest.

⬆️ Push

Add an element to the top of the stack. The top pointer increments by one, and the new element is placed at the new top position.

  • Time complexity: O(1)
  • top++ → stack[top] = element
  • Check for stack overflow first

⬇️ Pop

Remove and return the top element. The top pointer decrements by one. The element at the old top position is returned.

  • Time complexity: O(1)
  • element = stack[top] → top--
  • Check for stack underflow first

👁️ Peek

View the top element without removing it. The top pointer remains unchanged — a read-only operation.

  • Time complexity: O(1)
  • return stack[top] (no pointer change)
  • Useful for checking before pop
Design Choice

Why Stack for Digital Shadow?

The stack serves two critical features: Undo (reverse the last action) and Replay (play back a day in reverse order).

↩️ Undo Last Action

Every time the user adds an event, it's pushed onto the undo stack. When "Undo" is selected, the top event is popped and removed from the linked list — effectively reversing the last action.

  • Push: when event is added to linked list
  • Pop: when user selects "Undo"
  • After pop, find and delete from linked list
  • Single-level undo (most recent action only)

🔄 Replay Day

To replay a day's events in reverse, all events matching the date are pushed onto the stack. Then, popping them one by one yields the reverse chronological order — most recent first.

  • Filter linked list by target date
  • Push each matching event onto stack
  • Pop and display until stack is empty
  • Result: events from newest to oldest

⚡ O(1) Operations

Both push and pop are constant-time operations, making the undo and replay features extremely fast regardless of how many events are stored.

  • No traversal needed for push/pop
  • Array-based: direct index access
  • Predictable performance at any scale
  • Simple overflow/underflow checking

Operations

Stack Operations Step-by-Step

A detailed walkthrough of Push and Pop — the two fundamental operations that power undo and replay.

⬆️ Push Operation

1

Check Overflow

Verify top < MAX_SIZE - 1. If false, the stack is full — print an error and return.

2

Increment Top

Move the top pointer up: top = top + 1. This creates space for the new element.

3

Place Element

Copy the event data into the stack array: stack[top] = event. The element is now on the stack.

C — Push 
void push(Event *event) {
    if (top == MAX_SIZE - 1) {
        printf("Stack Overflow!\n");
        return;
    }
    top++;
    stack[top] = *event;
    printf("Pushed: %s\n", event->description);
}

⬇️ Pop Operation

1

Check Underflow

Verify top >= 0. If false, the stack is empty — print an error and return NULL.

2

Read Element

Copy the event data from the top: event = stack[top]. This captures the element before removal.

3

Decrement Top

Move the top pointer down: top = top - 1. The element is effectively removed from the stack.

C — Pop 
Event pop() {
    if (top == -1) {
        printf("Stack Underflow!\n");
        Event empty = {0};
        return empty;
    }
    Event popped = stack[top];
    top--;
    printf("Popped: %s\n", popped.description);
    return popped;
}

🔑 Key Insight

Events pushed in order 1, 2, 3, 4 come out as 4, 3, 2, 1. This natural reversal is exactly what we need for replaying a day's events — showing the most recent event first, then working backwards through the day to the earliest event. No sorting algorithm needed — the stack's LIFO structure handles it automatically.
