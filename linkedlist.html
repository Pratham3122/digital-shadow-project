Understanding Linked Lists

The backbone of Digital Shadow — a dynamic, pointer-based data structure that grows with your life's events.

Fundamentals

What is a Linked List?

A linked list is a linear data structure where each element (called a node) contains data and a pointer to the next node. Unlike arrays, linked lists are not stored in contiguous memory — each node can be anywhere in the heap.

🧱 Node

Each node is a self-contained unit with two parts: the data field (stores the event information) and the pointer field (stores the memory address of the next node).

🔗 Linking

Nodes are connected through pointers. The last node's pointer is set to NULL, indicating the end of the list. The first node is accessed through a head pointer.

📐 Non-Contiguous

Unlike arrays, nodes don't need to be next to each other in memory. Each node is allocated independently on the heap, connected only by the addresses stored in pointers.

Design Choice

Why Linked List for Digital Shadow?

We chose a singly linked list because life events are unpredictable — we never know how many events a user will log.

📈 Dynamic Size

Arrays require a fixed size at compile time. With a linked list, memory is allocated on demand — add 5 events or 5,000, the list adapts. No wasted space, no overflow errors.

  • No predefined MAX_EVENTS constant needed
  • Grows with each malloc(), shrinks with each free()
  • Memory used = exactly what's needed

⚡ Efficient Insertion & Deletion

Adding or removing an event from the middle of a linked list requires only pointer updates — no shifting of elements like in an array.

  • Insert at head: O(1) — just update head pointer
  • Delete any node: O(1) after finding it (pointer swap)
  • No element shifting like arrays require

🔄 Natural Event Ordering

Events are naturally sequential — each one follows another in time. A linked list's linear structure perfectly mirrors this chronological chain of life events.

  • Each node represents one moment in time
  • Traversal = walking through your timeline
  • Order maintained by insertion position

💾 Memory Efficiency

Each node allocates only the memory it needs. When events are deleted, memory is immediately returned to the system via free() — no fragmented unused slots.

  • malloc() allocates exactly sizeof(struct Event)
  • free() returns memory immediately
  • No "holes" in the data structure

Memory

Memory Allocation

Every node in the linked list is born through malloc() and dies through free() — the two fundamental heap operations in C.

🟢 malloc() — Allocate

The malloc() function requests a block of memory from the heap. It returns a pointer to the beginning of that block, or NULL if the system is out of memory.

C 
struct Event *newNode = (struct Event *)malloc(sizeof(struct Event));
if (newNode == NULL) {
    printf("Memory allocation failed!\n");
    return;
}

🔴 free() — Deallocate

The free() function returns the memory block to the heap. After freeing, the pointer becomes a dangling pointer — always set it to NULL after.

C 
free(targetNode);
targetNode = NULL; // Prevent dangling pointer

Pointers

Pointer Movement

Understanding how pointers change during linked list operations is essential. Here's what happens during an insertion at the tail.

1

Allocate New Node

Call malloc() to create a new node. The system returns a memory address, say 0x7A20. This becomes our newNode pointer.

2

Fill Node Data

Copy the event data into the new node's data field: strcpy(newNode->description, "Left Home"). Set newNode->next = NULL.

3

Traverse to Tail

Start at head and walk through the list: temp = head; while(temp->next != NULL) temp = temp->next;. Now temp points to the last node.

4

Link New Node

Set the tail's next pointer to the new node: temp->next = newNode. The previously NULL pointer now holds 0x7A20. The new node is part of the list.

Pointer Movement During Deletion

1

Find the Target Node

Traverse with two pointers: prev (one step behind) and curr (the current node). Stop when curr matches the search criteria.

2

Bypass the Node

Set prev->next = curr->next. This makes the predecessor point directly to the successor, effectively removing curr from the chain.

3

Free Memory

Call free(curr) to return the node's memory to the heap. Set curr = NULL to avoid dangling pointer issues.

Code

Node Structure in C

The C struct that defines each node in Digital Shadow's linked list. Every life event is stored in this structure.

C — Event Node Structure 
typedef struct Event {
    int    id;                    // Unique event identifier
    char   description[100];     // Event description text
    char   date[11];              // Date string "YYYY-MM-DD"
    char   time[6];               // Time string "HH:MM"
    char   category[30];           // Category: "Social", "Work", etc.
    struct Event *next;           // Pointer to next node
} Event;

// Global head pointer — entry point to the list
Event *head = NULL;

// Create a new event node
Event* createNode(int id, char* desc, char* date, char* time, char* cat) {
    Event *newNode = (Event *)malloc(sizeof(Event));
    if (newNode == NULL) {
        printf("Error: Memory allocation failed!\n");
        return NULL;
    }
    newNode->id = id;
    strcpy(newNode->description, desc);
    strcpy(newNode->date, date);
    strcpy(newNode->time, time);
    strcpy(newNode->category, cat);
    newNode->next = NULL;
    return newNode;
}

// Insert at the end of the list
void insertAtEnd(Event *node) {
    if (head == NULL) {
        head = node;
        return;
    }
    Event *temp = head;
    while (temp->next != NULL)
        temp = temp->next;
    temp->next = node;
}

📦 Struct Fields

  • id — Auto-incrementing integer for unique identification
  • description — Up to 100 characters describing the event
  • date — ISO format date string for consistent sorting
  • time — 24-hour format time string
  • category — Classification tag for analytics
  • next — Pointer to the next Event in the chain

📏 Memory Layout

Each Event node occupies approximately 152 bytes on a typical 64-bit system:

  • id: 4 bytes (int)
  • description: 100 bytes (char[100])
  • date: 11 bytes (char[11])
  • time: 6 bytes (char[6])
  • category: 30 bytes (char[30])
  • next pointer: 8 bytes (64-bit pointer)
  • Padding: ~1 byte (struct alignment)
